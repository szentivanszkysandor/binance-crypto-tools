Privacy Policy

Effective Date: September 30, 2025

1. Overview

The “Binance – Crypto Tools & Alert” Chrome extension provides a unified interface for monitoring live Binance Spot market data and using built-in, static HTML tools packaged with the extension (Order Book + Trades, Futures Sentiment, Taker Flow, EMA/RSI/MACD/ATR Monitor, Pattern Scanner, US Macro/Crypto Risk).

It displays real-time Price, RSI(14), 24h%, and an EMA Trend badge (20/50/200), and supports cross-based Price Alerts with configurable hysteresis and minimum repeat interval. The extension does not trade, manage wallets, or access private Binance accounts. It is independent and not affiliated with Binance. All data is informational and intended for educational/analytical purposes.

2. Permissions Used

  • storage — saves your configuration locally:
    • Selected Spot symbols and RSI timeframes; row order.
    • UI preferences (hide default tool buttons, hide Live Prices section, Compact View, active tab).
    • Column toggles (EMA 20/50/200, MACD, ATR%, BB width, 24h Volume) and minor UI state (e.g., dismissed tips).
    • Price Alerts (symbol, Above/Below, target, enabled, one-shot) and behavior (muteAlerts, hysteresis %, minRepeatMs, alertsStyle, hideAlerts in popup).
    • Local symbols cache for autocomplete (Binance Spot list; ~24h TTL).
  • notifications — shows a system notification only when a Price Alert fires (includes context like symbol, direction, target, latest price/distance; no personal data). Notifications offer action buttons (“Open on Binance”, “Disable alert”).
  • alarms — schedules lightweight background wake-ups so the service worker can periodically evaluate alerts and refresh minimal market data when the popup is closed. Not used for tracking or profiling.
  • offscreen — used solely to play a short alert sound reliably (in line with Chrome audio policies) even when the popup is closed. The offscreen document is created on demand and torn down afterwards; it does not capture content or user data.
  • Host permission — https://api.binance.com/* to fetch public, read-only market data:
    • Live prices/tickers and 24-hour stats.
    • Candlesticks (klines) for local indicator calculations (RSI/EMA/MACD/ATR).
    • Exchange info / Spot symbol list for autocomplete and validation.
    No authentication or account access; no trading/withdrawal endpoints are used.

The extension does not request access to browser tabs, browsing history, or other sensitive permissions.

2.1 Clipboard Use

When you click “Copy address” in the donation modal, the extension writes the shown BNB (BEP20) address to your clipboard. It does not read clipboard contents and never accesses the clipboard without your direct click.

3. Data Storage and Use

  • User settings: Stored in chrome.storage.sync (and may sync with your Google account if Chrome Sync is enabled) and in chrome.storage.local for caches.
  • No personal data: The extension does not collect email addresses, passwords, wallet keys, balances, or browsing history.
  • No external transmission: Apart from anonymous HTTPS requests to Binance public APIs, no data is sent anywhere. Nothing is shared with third parties.

4. Binance API Data

  • All requests go directly to official Binance endpoints and are read-only (no account login, no API keys).
  • Requests contain no personal identifiers. Results are processed locally to display prices/indicators and evaluate alerts.

5. Third-Party Services

  • Binance public APIs: sole source of market data. See Binance’s own policies for their services.
  • Donations: Optional Binance QR / PayPal links open external payment pages. The extension does not collect or store payment data.
  • Tools: All tool pages (Order Book, Taker Flow, etc.) are bundled static HTML within the extension, not remote web apps.

6. No Tracking or Analytics

  • No telemetry, analytics, or behavior tracking.
  • No cookies or fingerprinting.
  • No script injection into third-party webpages; the extension does not read page contents.

7. Incognito Mode

If you enable the extension for Incognito in Chrome, its privacy behavior remains the same as in normal mode.

8. Security

  • All data remains within Chrome’s storage areas controlled by the browser.
  • All external requests use HTTPS.
  • No backend servers; no handling of credentials or payment information.
  • No remote code execution; the extension does not fetch or execute code from external sources.

9. Data Retention & Deletion

  • Retention: Your symbol list, alerts, and preferences stay saved until you edit/reset them.
  • Manual management: Use the Options page to modify or clear your configuration at any time.
  • Uninstall: Removing the extension deletes its data from Chrome.

10. Changes

Any changes to this policy (e.g., due to feature or permission updates) will be reflected in the Chrome Web Store listing and the hosted policy page.

11. Contact

For support or privacy questions, please use the contact information provided on the Chrome Web Store listing.

12. Children’s Privacy

This extension is not directed to children under 13 and does not knowingly collect personal information from children.

13. No Financial Advice

The extension provides market data and tools for informational and educational purposes only. It does not constitute financial, investment, or trading advice. You are solely responsible for your trading decisions.

14. Data Practices Summary

  • Data collected: None (only local settings in Chrome storage).
  • Data shared / sold: None.
  • Purpose of network calls: Public market data from Binance only.

This extension complies with the Chrome Web Store Developer Program Policies and Google’s User Data policies. It collects no personal information and shares nothing with third parties.
