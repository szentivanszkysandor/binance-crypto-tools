Privacy Policy

Effective Date: September 25, 2025

1. Overview

The “Binance – Crypto Tools & Alert” Chrome extension provides a unified interface for monitoring live Binance Spot prices and using built-in HTML5 tools. It displays real-time Price, RSI(14), 24h%, and an EMA Trend (20/50/200) badge. It also includes cross-based Price Alerts with configurable hysteresis and minimum repeat intervals, optional audio cues, and multiple alert row display styles in the popup.

The extension does not trade, manage wallets, or access private Binance accounts. It is an independent, unofficial project and is not affiliated with Binance. All data is informational and intended for educational and analytical purposes only.

2. Permissions Used

  • storage
    Used to save your configuration. Two stores are used:
    • chrome.storage.sync — portable settings synced with your Google account if Chrome Sync is enabled.
    • chrome.storage.local — device-local cache and transient UI state.
  • notifications
    Used to display Chrome notifications when a price alert fires. The notification text contains only alert context (symbol, direction, target, latest price/distance); no personal data is included.
  • alarms
    Used to schedule low-frequency background wakeups so the service worker can refresh essential market data and evaluate alerts, even when the popup is closed. No user tracking or scheduling of personal events.
  • Host permission
    https://api.binance.com/* — strictly used to fetch public, read-only market data (prices, klines for RSI/EMA/MACD/ATR, 24h stats, and Spot symbol list for autocomplete). No authentication, no account endpoints.

The extension does not request access to tabs, browsing history, webRequest interception, or other sensitive permissions.

2.1 Clipboard Use

When you click a “Copy address” button in the donation dialog, the extension writes the shown BNB (BEP20) address to your clipboard. It does not read clipboard contents and never accesses the clipboard without your direct click.

2.2 Audio Playback

The packaged file assets/alert.wav is played locally for alert sounds when the popup/expanded window is open and you have interacted once (Chrome’s autoplay policy). Audio can be muted via the “Mute alert sound” setting. The extension does not record audio and does not use the microphone.

3. Data Stored by the Extension

  • In chrome.storage.sync (portable settings):
    • Live Prices list: your selected Spot symbols and their RSI timeframes, row order, and optional column toggles.
    • Price Alerts: per-row alert rules including { id, symbol, op (Above/Below), price (target), enabled, oneShot, createdAt }.
    • Alert behavior: muteAlerts, hystPct (hysteresis tolerance), minRepeatMs (cooldown), alertsStyle (render style), and hideAlerts.
    • UI prefs: show/hide defaults, hide Live Prices section, Compact View, active Options tab, etc.
  • In chrome.storage.local (device-local):
    • symbolsCache: the Binance Spot symbol list for autocomplete (timestamp + array), refreshed on demand or periodically.
    • Minor UI state: onboarding tip dismissal, open counters, other ephemeral flags.
  • Not stored: live prices, computed indicators, and alert fire events are processed in memory only and are not persisted.

4. Binance API Data

  • The extension fetches public, anonymous market data directly from Binance: live prices, candlesticks (for local RSI/EMA/MACD/ATR), 24-hour stats, and the Spot symbol list for autocomplete.
  • No Binance account login, API key, or authentication is required or used.
  • All requests are read-only and contain no personal identifiers.

5. How Alerts Work (Privacy-Relevant Summary)

  • Cross-based: an alert fires only when price crosses the target from the other side, considering your hysteresis setting.
  • Min repeat: enforces a cooldown between two fires of the same alert.
  • Once / Repeat: in 1× mode, an alert disables itself after the first fire; otherwise it re-arms after crossing back.
  • Notifications: the extension posts a Chrome notification with symbol and basic numbers; no personal data is included.
  • Audio: optional local playback of alert.wav in the popup/expanded window; can be muted.

6. Third-Party Services

  • Binance APIs — market data comes directly from Binance. Refer to Binance’s own policies for their services.
  • Developer-hosted tools — the “Tools” buttons open static HTML pages (e.g., GitHub Pages). These pages do not receive data from the extension and operate independently, subject to their hosting provider’s policies.
  • Donations — optional Binance QR and PayPal links open external payment pages. The extension does not collect or store any payment information.

7. No Tracking or Analytics

  • No telemetry or usage analytics.
  • No cookies or fingerprinting.
  • No content script injection into third-party websites and no reading of web page content.

8. Incognito Mode

The extension may be allowed to run in Incognito if you enable it in Chrome. Its privacy behavior is identical to normal mode.

9. Security

  • All settings remain within Chrome’s extension storage (sync/local).
  • All network requests use HTTPS.
  • No backend servers; no handling of sensitive credentials or payment data.
  • No remote code execution; the extension does not fetch or execute code from external sources.
  • Notifications are delivered by Chrome/OS; their visibility on lock screens depends on your device settings.

10. Data Retention & Deletion

  • Retention: your symbol list, alert rules, and preferences persist until you change or reset them.
  • Manage locally: use the Options page to edit or clear your configuration at any time.
  • Uninstall: removing the extension deletes its data from your browser. If Chrome Sync is enabled, clearing synced data is managed by your Google account settings.

11. Changes

If this policy changes due to feature updates or permission changes, the latest version will be available on the Chrome Web Store listing.
